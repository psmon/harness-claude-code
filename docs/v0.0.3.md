## 핵심 설계 결정
1. **Agentic Loop을 Journey 상태 모델에 오버레이** — 기존 선형 Journey(idle→prompted→planning→building→recording)를 유지하면서, planning/building/recording 각각에 Agentic Loop의 3단계(Gather Context/Take Action/Verify Results)를 매핑. 기존 모델을 깨지 않으면서 반복 실행 패턴을 도입.
2. **에이전트 3종 역할 분리** — architect(설계), journey-recorder(기록), memory-curator(지식)로 관심사를 분리. 각 에이전트가 Agentic Loop의 어느 단계에서 역할하는지 명시.
3. **오케스트레이터 패턴 3종** — Chain(기본), Parallel(지식 보강), Verify(검증 실패 시 복귀). 상황에 따라 유연하게 선택.
4. **실행 모드 선택 기준 정의** — 하네스 업그레이드/단순 추가 = 시니어 에이전트 모드, 오픈소스 학습/다계층 변경 = 에이전트 팀 모드.
