## revfactory 하네스의 핵심 특징

### 도메인 분석에서 출발하는 6-Phase
1. **도메인 분석** → 2. 팀 디자인 → 3. 에이전트 정의 → 4. 스킬 설계 → 5. 워크플로우 → 6. 검증 & 배포

기존 bkit이 PDCA(Plan-Do-Check-Act) 사이클 중심이라면, revfactory는 **도메인 분석이 선행**되는 점이 차별화됨. 문제 영역을 먼저 이해한 후 팀과 에이전트를 구성한다는 접근.

### Core Components
- **오케스트레이터**: 에이전트들을 조율하는 중앙 컨트롤러
- **에이전트 A/B/C**: 각각 스킬을 보유한 독립 에이전트
- **SharedMemory & ContextWindow**: 에이전트 간 상태 공유

### Two Execution Modes
- **에이전트 팀 모드**: SpawnManager + TaskCreate 기반 (기본/실행)
- **시니어 에이전트 모드**: 메인 Agent 중심 조율, 글로벌 컨텍스트

### Architecture Patterns & Data Protocols
- 패턴: Chain, Verify, Parallel, Workflow
- 프로토콜: SendMessage(실시간), TaskCreate/Update(추적), 파일 기록(Workspace)
