# 하네스 작성 스킬 가이드

> `harness-create` 스킬을 활용하여 하네스의 부분 요소를 만드는 베스트 프랙티스

## 스킬 소개

`harness-create`는 Claude Code Skills 2.0 기반의 프로젝트 전용 스킬입니다.
하네스의 하위 요소를 생성하거나 하네스 자체를 업그레이드할 때 프로젝트 규칙을 자동으로 가이드합니다.

### 발동 조건
- `/harness-create` 슬래시 커맨드로 직접 호출
- "하네스를 추가 개선 하고자 합니다" 등 하네스 개선 관련 대화에서 자동 발동
- 새 에이전트, 지식, 엔진 컴포넌트 추가 요청 시

### Skills 2.0 적용 기능
| 기능 | 활용 |
|------|------|
| 동적 컨텍스트 주입 | harness.config.json, 최신 버전, git log를 자동 로딩 |
| Progressive Disclosure | reference/ 파일을 필요 시에만 로딩 |
| 문자열 치환 | $ARGUMENTS로 작업 내용 전달 |
| 도구 제한 | memorizer/pencil MCP 도구 자동 허용 |

## 시나리오별 베스트 프랙티스

### Scenario A: 새 에이전트 추가

**프롬프트 예시**:
```
/harness-create architect 에이전트를 추가하고 싶습니다
```

**권장 진행**:
1. 스킬이 6-Phase 워크플로우를 안내합니다
2. Phase 1(도메인 분석): memorizer에서 관련 에이전트 패턴 검색
3. Phase 3(에이전트 정의): `agents/architect.md` 스펙 작성
4. Phase 6(검증): 4종 산출물 갱신 확인

**핵심 포인트**:
- 기존 `agents/TODO.md`에 정의된 계획 에이전트(architect, journey-recorder, memory-curator)를 참고
- 최소 에이전트 원칙: 하나씩 추가하고 검증

### Scenario B: 하네스 업그레이드

**프롬프트 예시**:
```
하네스를 추가 개선 하고자 합니다. hooks 시스템을 구현하려 합니다.
```

**권장 진행**:
1. 스킬이 업그레이드 워크플로우를 안내합니다
2. 현황 파악 → 계획 수립 → 구현 → 산출물 갱신
3. 특히 `docs/` 최신 버전의 "다음 버전 예정"을 확인하여 연속성 유지

**핵심 포인트**:
- Plan 모드를 적극 활용하여 설계 먼저
- 구현 후 반드시 4종 산출물 체크리스트 완료

### Scenario C: 현황 파악

**프롬프트 예시**:
```
/harness-create 현재 하네스 구조를 설명해주세요
```

**권장 진행**:
1. 3계층 가이드를 기반으로 현재 구조 분석
2. 각 디렉토리의 TODO.md로 계획/진행 상황 파악
3. 블루프린트 스크린샷으로 시각적 현황 제공

---

## 4종 산출물 갱신 순서

산출물을 갱신할 때 권장하는 순서:

1. **docs/vX.Y.Z.md** (먼저) — 작업 내용을 정리하며 여정을 기록
2. **design/architecture.md** — mermaid 다이어그램에 구조적 변경 반영
3. **design/blueprint.pen** — Pencil MCP로 시각적 블루프린트 갱신
4. **README.md** (마지막) — 버전 테이블 행 추가, 개념 갱신

> 이 순서는 "기록 → 설계 → 시각화 → 공개"의 흐름을 따릅니다.

## 프롬프트 작성 팁

### 효과적인 프롬프트 구조

```
[작업 유형] + [대상] + [목적/이유]
```

**좋은 예**:
- "architect 에이전트를 추가하고 싶습니다. 설계 리뷰와 블루프린트 갱신을 자동화하기 위해서입니다."
- "하네스를 개선하고 싶습니다. hooks 시스템을 구현하여 SessionStart 이벤트를 처리하려 합니다."
- "engine의 워크플로우에 Verify 패턴을 추가하고 싶습니다."

**피해야 할 예**:
- "뭔가 추가해주세요" (대상 불명확)
- "하네스 고쳐주세요" (목적 불명확)

### $ARGUMENTS 활용

`/harness-create` 뒤에 오는 인자가 `$ARGUMENTS`로 전달됩니다:
```
/harness-create [여기가 $ARGUMENTS]
```

구체적일수록 스킬이 정확한 워크플로우를 선택합니다.

## 주의사항

1. **blueprint.pen 접근**: 반드시 Pencil MCP 도구(`mcp__pencil__*`)로만 접근. Read/Grep 사용 금지
2. **memorizer 검색**: 한글/영문 키워드 모두 시도하여 누락 방지
3. **버전 연속성**: 최신 `docs/vX.Y.Z.md`의 "다음 버전 예정"을 확인하여 작업의 연속성 유지
4. **최소 변경 원칙**: 필요한 부분만 변경, 과도한 리팩토링 지양
5. **TODO.md 갱신**: 새 요소가 추가된 디렉토리의 TODO.md도 함께 갱신
