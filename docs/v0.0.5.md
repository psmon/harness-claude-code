# v0.0.5 - Playwright 검증 워크플로우 추가

**날짜**: 2026-03-23
**프롬프트**: sample1 프로젝트 Playwright CLI 검증 후 엔진 개선

## 이 버전에서 한 일
- engine/에 `playwright-verification.md` 추가 — Playwright CLI 기반 브라우저 검증 워크플로우
- Journey 상태 모델에 `verifying` 상태 추가 (building → verifying → recording)
- `project-creation-workflow.md`에 Phase D(Playwright 검증) 삽입
- 검증 대상: Chromium Desktop (1280x720) + Chromium Mobile (375x812), Firefox 제외

## 핵심 설계 결정
1. **verifying 상태 도입** — 기존 `building → recording` 사이에 `verifying` 상태를 추가. 정적 분석만으로는 발견할 수 없는 크로스브라우저·반응형 이슈를 실제 브라우저 렌더링으로 포착.
2. **Chromium 전용** — Firefox는 Babel standalone `type="text/babel"` 외부 스크립트 호환성 이슈로 CDN 기반 프로젝트에서 안정적 검증 불가하여 제외. 빌드 도구 기반 프로젝트에서는 추후 확장 가능.
3. **Verify Loop 패턴** — 검증 실패 시 building으로 복귀하여 수정 후 재검증하는 루프 패턴 적용. Agentic Loop의 Verify Results가 실제 도구(Playwright)와 연결됨.

## 여정 기록

### 작업 과정
1. projects/sample1에 대해 Playwright CLI로 3개 브라우저/뷰포트 스크린샷 캡처
2. Chromium Desktop: PASS / Firefox: FAIL(텍스트 미렌더링) / Chromium Mobile: FAIL(레이아웃 깨짐) 확인
3. 정적 분석(v0.0.4 체크리스트)에서 4/4 PASS였던 결과가 실제 검증에서 3/4 PASS로 변경
4. 이 경험을 engine/ 워크플로우로 체계화

### harness에 반영한 점
- `engine/` — Playwright 검증 워크플로우 신규 추가, Journey 상태 모델 확장
- `engine/` — project-creation-workflow.md 6단계로 확장 (Phase A~F)
- 교훈: 정적 분석만으로는 실제 렌더링 이슈를 놓칠 수 있음. 브라우저 검증은 필수.
