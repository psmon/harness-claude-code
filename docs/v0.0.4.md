## 핵심 설계 결정
1. **PRD → 디자인 → 코드 파이프라인 확립** — memorizer PRD 메모리를 시작점으로, Pencil 디자인을 거쳐 코드를 생성하는 표준 파이프라인 정의. 디자인 파일의 색상/타이포를 코드에 충실히 반영.
2. **프로젝트 디자인 분리** — 프로젝트별 디자인은 `design/project1.pen`으로 하네스 블루프린트(`design/blueprint.pen`)와 분리 관리.
3. **harness-usage.md 표준화** — 프로젝트 생성 시 반드시 하네스 활용 기록을 남기도록 표준 양식 정의. 하네스 개선 피드백 루프의 핵심.
4. **CDN 기반 React** — 빌드 없이 브라우저에서 직접 실행 가능한 CDN 방식 채택 (React 18 + Tailwind CSS + Babel Standalone).
