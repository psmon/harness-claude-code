# engine/ - 여정 엔진 (상태 관리)

## 역할
프롬프트 기반 개발 여정의 상태를 추적하고 관리합니다.
bkit의 PDCA State Machine 대신, Journey(여정) 기반 상태 모델을 사용합니다.

## 개발 예정
- [ ] Journey 상태 모델 정의 (prompt -> plan -> build -> record)
- [ ] 버전 넘버링 자동화 (v0.0.x)
- [ ] 프롬프트-결과 매핑 추적
- [ ] 블루프린트 변경 이력 관리
- [ ] 여정 타임라인 생성기

## 참고
- bkit lib/pdca/ 상태 머신 패턴 참고
- 단, PDCA의 복잡한 가드/트랜지션 대신 단순한 선형 여정 모델로 시작
